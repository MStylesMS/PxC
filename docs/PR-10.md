# PR-10: CI Pipeline and .ini Auto‑Loader

Author: Team Paradox  
Date: 2025-08-11  
Status: Proposal  
Scope: Add GitLab CI pipeline and introduce a build/runtime .ini-driven configuration autoloader without ejecting CRA.

## Summary
This PR proposes two infrastructure improvements:

1) GitLab CI pipeline to lint, test, and build the app on every push/MR, with build artifacts.
2) A .ini auto-loader that generates a public runtime config (config.js) from `config/{env}.ini` during prestart/prebuild, so environments can be switched via .ini files or env vars without code changes.

No user-visible UI changes; minimal code wiring in `src/` to read config.

---

## Motivation
- Consistent quality gates (lint/test) and reproducible builds in CI.
- Simpler configuration management across environments (dev, staging, prod) using human-readable `.ini` files and safe CRA-compatible runtime injection.

---

## Design Overview

### A) GitLab CI Pipeline
- Runner image: `node:18-bullseye` (align with dev runtime).
- Stages: `lint` → `test` → `build`.
- Cache `node_modules` keyed by `package-lock.json` to speed up jobs.
- Artifacts: `build/` attached to the `build` job for download or later deploy jobs.

Proposed `.gitlab-ci.yml` (high level):

```yaml
image: node:18-bullseye

cache:
  key:
    files:
      - package-lock.json
  paths:
    - node_modules/

stages: [lint, test, build]

variables:
  CI: "true"
  # Optional CRA variables; can be overridden in GitLab CI variables per-environment
  REACT_APP_MQTT_HOST: "localhost"
  REACT_APP_MQTT_PORT: "1884"

install:
  stage: lint
  script:
    - npm ci
  artifacts:
    paths: [node_modules/]
    expire_in: 1h

lint:
  stage: lint
  needs: [install]
  script:
    - npm run lint

unit_tests:
  stage: test
  needs: [install]
  script:
    - npm test -- --watchAll=false
  coverage: "/All files[^\n]*\n\s*Lines\s*:\s*([0-9.]+)%/"

build:
  stage: build
  needs: [install]
  script:
    - npm run build
  artifacts:
    paths:
      - build/
    expire_in: 7 days
  rules:
    - if: $CI_PIPELINE_SOURCE == "merge_request_event"
    - if: $CI_COMMIT_BRANCH
```

Notes:
- We keep jobs simple and fast; deployments can be added later as a follow-up PR (e.g., static host, Docker, or kiosk packaging).
- `coverage` regex is optional and tuned for CRA/Jest output; adjust as needed.

### B) .ini Auto‑Loader (CRA‑friendly)
Goal: Load config from `config/{env}.ini` and expose it to the browser at runtime, without ejecting CRA.

Approach:
- Add a Node pre-script `scripts/prepare-config.js` that:
  - Selects environment: `CONFIG_ENV || NODE_ENV || 'development'`.
  - Reads and parses `config/${env}.ini` using the `ini` package (already in dependencies).
  - Validates keys and falls back to sensible defaults.
  - Writes `public/config.js` with `window.__APP_CONFIG__ = { ... }`.
- Add a script tag in `public/index.html` to load `%PUBLIC_URL%/config.js` early.
- Create `src/config.js` helper to read `window.__APP_CONFIG__` and merge CRA env overrides (`REACT_APP_*`).
- Update `src/MQTT.js` to use values from `src/config.js` instead of hardcoded defaults.

Configuration precedence:
1. CRA env vars (REACT_APP_MQTT_HOST/PORT/TOPIC, etc.).
2. `public/config.js` (generated from `config/{env}.ini`).
3. Safe hardcoded defaults as last resort.

Validation:
- Warn and continue on missing keys with defaults.
- Fail the build if `config/${env}.ini` is missing unless `ALLOW_MISSING_INI=true` is set (default: fail to catch misconfigurations early).

Security:
- Only non-secret UI config should live in `.ini`/`config.js`. Secrets stay server-side.

Examples (illustrative, exact code to be added in implementation PR):

`scripts/prepare-config.js` outline:
```js
#!/usr/bin/env node
const fs = require('fs');
const path = require('path');
const ini = require('ini');

const env = process.env.CONFIG_ENV || process.env.NODE_ENV || 'development';
const iniPath = path.resolve(__dirname, `../config/${env}.ini`);
if (!fs.existsSync(iniPath) && process.env.ALLOW_MISSING_INI !== 'true') {
  console.error(`[config] Missing ${iniPath}. Set CONFIG_ENV or provide an .ini.`);
  process.exit(1);
}
let cfg = {};
if (fs.existsSync(iniPath)) {
  cfg = ini.parse(fs.readFileSync(iniPath, 'utf-8'));
}
const out = `window.__APP_CONFIG__ = ${JSON.stringify(cfg)};\n`;
fs.writeFileSync(path.resolve(__dirname, '../public/config.js'), out);
console.log(`[config] Wrote public/config.js for env: ${env}`);
```

`src/config.js` outline:
```js
// Merge CRA env overrides with generated config
const raw = (typeof window !== 'undefined' && window.__APP_CONFIG__) || {};
const fromEnv = {
  mqtt: {
    host: process.env.REACT_APP_MQTT_HOST,
    port: process.env.REACT_APP_MQTT_PORT && Number(process.env.REACT_APP_MQTT_PORT),
    topic: process.env.REACT_APP_MQTT_TOPIC,
  },
};
function pick(a, b, fallback) { return a ?? b ?? fallback; }
export const appConfig = {
  mqtt: {
    host: pick(fromEnv.mqtt.host, raw?.mqtt?.host, 'localhost'),
    port: pick(fromEnv.mqtt.port, raw?.mqtt?.port, 1884),
    topic: pick(fromEnv.mqtt.topic, raw?.mqtt?.topic, 'Paradox/Houdini/Mirror/Clock/Commands'),
  },
  display: {
    fade_duration_default: pick(raw?.display?.fade_duration_default, undefined, 2000),
    hint_duration_default: pick(raw?.display?.hint_duration_default, undefined, 25),
    clock_orientation: pick(raw?.display?.clock_orientation, undefined, -90),
  },
};
```

`public/index.html` addition:
```html
<!-- Load runtime config before the app bundle -->
<script src="%PUBLIC_URL%/config.js"></script>
```

`package.json` script hooks:
```json
{
  "scripts": {
    "prestart": "node scripts/prepare-config.js",
    "prebuild": "node scripts/prepare-config.js"
  }
}
```

## Acceptance Criteria
- CI runs `lint`, `test`, and `build` on pushes and MRs; build artifacts are downloadable.
- `npm start` and `npm run build` automatically generate `public/config.js` from the selected `.ini` (development by default).
- App reads MQTT host/port/topic via `src/config.js` with env overrides; defaults preserved if missing.
- Documentation updated to explain configuration precedence and how to add new environments.
- No ejection from CRA; local dev workflow remains `npm start`.

## Risks & Mitigations
- Missing `.ini` → build fail: mitigated with clear error and `ALLOW_MISSING_INI=true` escape hatch.
- Caching `node_modules` issues → cache keyed by lockfile; safe to purge if needed.
- New `config.js` not found → guard with defaults in `src/config.js` and document script tag placement.
